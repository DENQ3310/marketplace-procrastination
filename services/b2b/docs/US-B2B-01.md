# US-B2B-01: создание карточки товара

## Реализация

`POST /api/v1/products` создаёт карточку товара со статусом `CREATED`.
`seller_id` берётся только из JWT, переданное клиентом поле игнорируется.
Категория проверяется до создания товара. Изображения обязательны и вместе с
характеристиками сохраняются в одной транзакции с товаром. Создание товара не
отправляет событие в Moderation, пока к нему не добавлен первый SKU.

Ошибки валидации возвращаются со статусом `400` в едином формате:

```json
{"code": "VALIDATION_ERROR", "message": "Invalid request fields: images"}
```

Несуществующая категория возвращает:

```json
{"code": "INVALID_CATEGORY", "message": "Category not found"}
```

## Тесты

Сценарии находятся в `tests/integration/test_create_product.py`:

- `create_product_returns_201_with_created_status`;
- `seller_id_taken_from_jwt`;
- `missing_images_returns_400`;
- `missing_category_returns_400`;
- `invalid_category_id_returns_400`.

## ADR

Рассматривались JSON-поле в `Product`, отдельная таблица характеристик и
классическая EAV-схема. Выбрана отдельная таблица `Characteristic`: она проще
EAV в сопровождении и позволяет фильтровать характеристики средствами SQL без
разбора JSON. Новые характеристики добавляются без изменения схемы таблицы
товаров, а связь с `product_id` или `sku_id` позволяет использовать одну модель
для характеристик товара и его вариантов.
